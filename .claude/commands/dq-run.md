DQ 파이프라인을 실행하고 결과를 보고합니다.

## 실행 규칙

인자($ARGUMENTS)를 파싱합니다:
- 첫 번째 토큰이 존재하면 입력 파일 경로로 사용합니다.
- 파일이 지정되지 않으면 프로젝트 루트에서 가장 최근에 수정된 JSON / JSONL / CSV 파일을 자동으로 찾아 사용합니다.
- `--batch-size N` 이 포함되면 해당 값을 사용하고, 없으면 기본값 100 을 사용합니다.
- `--skip-openai` 가 포함되면 해당 플래그를 전달하고, 없으면 기본으로 포함합니다.

## 실행 순서

1. 위 규칙으로 파일과 옵션을 결정한 뒤 사용자에게 한 줄로 확인합니다.
   예) `customer_data_quality_test_50.json` 을 batch-size=100 로 실행합니다.

2. Bash 도구로 아래 명령을 실행합니다:
   ```
   cd <프로젝트 루트> && python main.py <파일> --batch-size <N> --skip-openai
   ```

3. 실행이 완료되면 다음 항목을 요약합니다:
   - 소요 시간
   - 최종 DQ 점수 / 등급
   - Critical 필드 수, Warning 필드 수
   - 생성된 보고서 경로

4. "결과를 자세히 보려면 `/dq-result`, 비용 분석은 `/dq-cost` 를 실행하세요." 를 안내합니다.
