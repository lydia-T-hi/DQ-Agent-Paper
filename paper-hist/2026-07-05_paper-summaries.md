## 1. 논문 계획 요약

> 근거 문서: 04_experiment-plan_v1_1.md, 05_agent-architecture_v1_1.md.
> **상위 문서 00_thesis-design_v4가 저장소에 없어** 아래는 하위 문서에서 역추적한 재구성임 —
> 00 문서를 저장소에 추가하고 이 요약을 대조·수정할 것.

**주제**: LLM 비용-효율적 데이터 품질(DQ) 자동 검사를 위한 단계형 멀티에이전트 파이프라인

**핵심 주장**: 모든 레코드를 LLM에 보내는 대신, (1) 결정론적 프로파일링·정규화로
명백한 오류를 선처리하고 (2) 통계적으로 모호한 셀만 LLM(Detector)에 선택 라우팅하며
(3) 이종 모델 Judge(Validator)와 분포 기반 검증(HC)으로 LLM 판단을 교차검증하면,
탐지 성능(F1)을 유지·개선하면서 토큰 비용을 크게 절감할 수 있다.

**연구 질문** (04 §5 가설에서 역추적):
- RQ1: 선택적 LLM 호출(A3)이 결정론적 처리만(A1)보다 탐지 성능을 유의하게 높이는가? (H1: F1(A3)>F1(A1))
- RQ2: 검증 스테이지 추가(A5)가 성능을 유지·개선하는가? (H2: F1(A5)≥F1(A3))
- RQ3: 증분 비용 대비 성능 이득(CEI)은 어느 스테이지에서 가장 큰가? (H3: CEI(2B증분)>CEI(3A증분))

**기여점(안)**:
1. 결정론/LLM 역할 분리 기반 5단계 DQ 파이프라인 아키텍처와 상태 계약
2. 셀 기준 오류 주입 프로토콜(유형 5종×오류율×시드)과 사전 등록식 실험 설계
3. 비용-효율 지표(CEI)에 의한 구성별 한계 효용 분석 — 토큰·USD 이중 산출
4. 한국어 필드 포함 합성 데이터(S1)로 오염 통제 + 국문 학술지 독자 적합성

## 2. 실험 계획 요약 (04 문서 → 논문 실험 장 구조)

- **3단 구조**: E0 파일럿(지표 검증, 논문 미포함) → E1 주실험(S1·S2 합성, A1~A5×ρ{10,20}%×시드5, 100회)
  → E2 보조실험(Hospital·Flights 벤치마크, 외부 도구 비교)
- **오염 통제**: S1·S2는 생성 시점 신규 값 → LLM 사전학습 포함 불가능. E2는 오염 가능성
  명시 후 "F1 상한 추정치"로만 보고
- **공정성 프로토콜**: A2(전 행 LLM)와 A3(선택 호출)에 동일 프롬프트·few-shot k=3·배치 20행/호출.
  모델·temperature 고정, 프롬프트 전문 부록 공개
- **지표**: 셀 단위 F1(유형별 분해), Consistency(고정 배치 5회 재실행 일치율 + Fleiss' κ),
  Cost(토큰 주/USD 보조), CEI = ΔF1/ΔCost
- **검정**: Wilcoxon signed-rank(시드 쌍대) + Cliff's delta, Holm 보정, α=0.05. 가설 사전 등록,
  기각 시에도 그대로 보고
- **신뢰성 장치**: 실패 시 1회 재시도 후 결측 기록(silent retry 금지), 파싱 실패도 F1에 계수,
  실행 중 코드 수정 시 전체 재실행, E1 시작 시 커밋 해시 동결

## 3. 선행연구 요약 (관련연구 장 초안)

### 3.1 규칙·통계 기반 오류 탐지

**Raha** (Mahdavi et al., SIGMOD 2019) — [검증: 공식 저장소 github.com/BigDaMa/raha 확인됨]
설정 불필요(configuration-free) 오류 탐지 시스템. 다수의 탐지 전략(패턴·규칙·이상치 기반)을
피처로 삼아 소량의 사용자 라벨로 분류기를 학습. Hospital·Flights 등 표준 벤치마크 확립.
→ 본 논문과의 관계: E2 벤치마크 출처이자 "비LLM 최강 베이스라인" 위치. 우리는 라벨 대신
LLM 판단을 쓰되 선택 호출로 비용을 통제한다는 차별점.

**Baran** (Mahdavi & Abedjan, PVLDB 2020) — [모델 지식 — 원문 대조 필요]
Raha의 오류 **수정** 대응물. 오류 문맥에서 수정 모델을 학습하고 전이. → 우리 2B의
corrected_value 제안과 비교 지점.

**HoloClean** (Rekatsinas et al., PVLDB 2017) — [모델 지식 — 원문 대조 필요]
무결성 제약·통계 신호를 확률 그래프 모델로 통합해 확률적 복구 수행. → 규칙(R1~R8)과
통계(Z-score)를 합의로 통합하는 우리 Stage 4 설계의 고전적 선례.

### 3.2 LLM 기반 데이터 정제

**Can Foundation Models Wrangle Your Data?** (Narayan et al., PVLDB 2022) — [모델 지식 — 원문 대조 필요]
대형 언어모델 few-shot 프롬프팅만으로 오류 탐지·엔티티 매칭 등 정형 데이터 작업에서
경쟁력 있는 성능을 보임을 처음 체계적으로 제시. → "LLM이 DQ에 유효하다"는 출발점.
단, 전 레코드 호출 비용 문제는 미해결 — 본 논문의 공백 지점.

**ZeroED** (arXiv:2504.05345, 2025) — [검증: arXiv ID 확인됨, 내용은 abstract 수준]
LLM 추론과 경량 학습을 결합한 하이브리드 zero-shot 오류 탐지. → 최신 LLM-DQ 계열로
관련연구에 포함 필요. 비용 계측·구성별 한계 효용 분석은 다루지 않음(차별점).

### 3.3 LLM-as-a-Judge와 이종 모델 검증 (3A Validator 근거)

**Judging LLM-as-a-Judge with MT-Bench and Chatbot Arena** (Zheng et al., NeurIPS 2023
Datasets & Benchmarks) — [모델 지식 — 원문 대조 필요]
강한 LLM 심판이 인간 선호와 80%+ 일치함을 보였으나 위치 편향·장황함 편향·자기선호
편향을 보고. → Judge 유효성의 근거인 동시에 편향 목록의 출처.

**LLM Evaluators Recognize and Favor Their Own Generations** (Panickssery et al., NeurIPS 2024)
— [모델 지식 — 원문 대조 필요]
LLM이 자기 출력을 식별·선호하는 자기선호 편향을 실증. → **2B(Claude) 판단을 동일 모델이
아닌 GPT(3A)로 교차검증하는 이종 모델 설계의 직접 근거** (05 §2.5).
