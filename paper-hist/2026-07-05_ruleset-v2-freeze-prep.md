## 3. DQ 점수 산정 룰 v2 — ISO/IEC 25012 차원 기반 (`config/dq_scoring.json`)

기존 문제: 소스별 가중(40/35/25)이 3A 유무에 따라 재분배(60/40)되어 **구성 간 산식이 달랐음**.

개정: 차원 점수 = 100×(1−영향레코드/전체), 종합 = 차원 가중합
(완전성 .25 / 유효성 .25 / 정확성 .20 / 일관성 .15 / 유일성 .15 — DAMA 계열 관행).
규칙→차원 매핑 포함, **산식은 config와 무관하게 동일**. 단 점수는 '탐지된' 위반 기반이므로
구성 비교는 여전히 F1/비용으로만 (04 §5 유지).

## 5. 보완 사항 (판단에 의한 갭 필링)

1. **2A 모호성 판정 전면 교체 — 강건 통계.** 주입 데이터 스모크에서 ambiguous=0건 발견:
   10^12 극단값이 평균·표준편차를 부풀려 자신의 Z-score를 3 이하로 마스킹 →
   **A3≡A1이 되어 H1 검정 자체가 불가능해지는 치명 결함.**
   중앙값/MAD 기반 수정 Z-score(>3.5, Iglewicz & Hoaglin) + 교차필드 날짜쌍 모순으로 교체
   (05 §2.3 기준의 구현). 교체 후 동일 데이터에서 ambiguous 99건.
2. **3A 입력 샘플링**: 전 레코드 전송 → 1,000행이면 gpt-4o 컨텍스트 초과.
   changelog 레코드 우선 + 순차 보충, 상한 80건 (결정론적 — 무작위 없음)
3. run_id에 rep 병기 — 동일 시드 반복(Consistency) 로그 덮어쓰기 방지
